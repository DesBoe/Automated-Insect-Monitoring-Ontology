# Automated-Insect-Monitoring-Ontology
Working towards a data standard or data schema for devices that try to image arthropods automatically in nature by creating a shared vocabulary.

> [!NOTE]
> Nerds who are good at computers and organizing stuff might have better suggestions for formatting this list of terms and definitions. If that's you, please change this however makes sense!

## Glossary of terms

### Creature
- **Definition:** Abstract term for a living thing that is of interest for this work.
- **Explanation:** Could be an insect, an arthropod, or another small animal.
- **Example:** gecko, beetle, bat

### Detection
- **Definition:** a specific subset from a source file.
- **Explanation:** ideally contains only one creature.
- **Example:** cropped image of an insect
- **Attributes:**
	- Date and time stamp
	- Source file name
	- Information on bounding boxes (e.g. X & Y coordinates + dimensions) or start timestamp + length
	- Taxa (e.g. kingdom, class)
	- Source image coordinates (center and bounding box)
	- Human_ID
	- Machine_ID
	- Model name (model used for localization and/or determination)

### Track
- **Definition:** A time sequence of continuous detections.
- **Explanation:** multiple and subsequent detections showing the same taxa.
- **Example:** a beetle walking over a screen, a bumble bee visiting close by flowers
- **Attributes:**
    - duration

### Source File
- **Definition:** A media file that can have different types of backgrounds and may contain one or more target species.
- **Explanation:** any computer readable file format generated by the machine that might contain unprocessed detections
- **Example:** photograph, video, or audio track (or combined).
- **Assumptions:**
	- Static device. 

### Session
- **Definition:** A continuous series of source files.
- **Explanation:** the result of one observation period.
- **Example:** Source files captured during one night (dusk to dawn), one day (afternoon) or only during dusk

### Deployment
- **Definition:** A collection of sessions.
- **Explanation:**Starts when a device is left in a location and ends when the device is removed from that location.
- **Example:** A seaon

### Device
- **Definition:** A machine that performs some kind of automated observation task.
- **Explanation:** Hardware that is deployed in the location
- **Example:** all great devices listed in [InsectAI hardware-db](https://github.com/InsectAI-COST-Action/hardware-db)
- **Attributes:**
	- Camera + lens or/and microphone used
	- Orientation (vertical vs horizontal)
	- Effective resolution (e.g. px/mm, kbits/sec)
	- Sensitivity range (IR, RGB, UV, infrasonic, ultrasonic)
	- Unique machine_ID

### Target
- **Definition:** The part of a device being directly observed where insects are intended to go for observation.
- **Explanation:** 
- **Example:** a target sheet, a tube, or an artificial flower.

### Attractor
- **Definition:** specific stimulus used to lure creatures.
- **Explanation:** This might not be visible/ audible in the Source files.
- **Example:**chemical (smell), visual (UV, false flowers), or auditory (mating call).

### Device Location
- **Definition:** A very specific spot where a device is deployed.
- **Explanation:** includes all information about the exact spot of the device
- **Example:** 5.094, -80.34; left palm tree next to lake"; 5m above ground
- **Attributes:**
    - GPS coordinate 
    - a short description of the exact spot.
    - height above ground

### Site
- **Definition:** The area being surveyed by a device.
- **Explanation:** includes all information about the area surrounding the device. Might include a GPS coordinate and a radius. Automated Insect Monitoring Symposium participants indicated this is likely a zone 10–100 meters wide.
- **Example:** flood plain forest, 100m around device, trees up to 30m
- **Attributes:** 
    - Canopy height
    - Habitat type

### Protocol
- **Definition:** The method in which a task was carried out.
- **Explanation:** The description needs to include all necessary information to deploy the device in the same maner for a follow up survey.
- **Example:** checklist, bullet points, might include photodocumentary. Can be applied to a deployment, session, capture, or detection.

### Annotation
- **Definition:** assigned species/taxon name of a detection
- **Explanation:** can be used in training data sets or while/after validating AI results
- **Example:** Apis sp. (instead of bee)
- **Attributes:**
    - name (of anotator)
    - reference species catalogue (to avoid synonyms)

### Terms without definition (yet) that might help to describe or distinguish things
- appearance
- localisation
- record
